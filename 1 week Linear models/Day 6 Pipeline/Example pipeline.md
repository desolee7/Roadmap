# Реализация Pipeline в sklearn: Transform, FunctionTransformer, ColumnTransformer и другое

## Общая идея

В sklearn пайплайн строится из последовательности шагов обработки данных и модели.  
Каждый шаг должен быть либо:
- transformer (имеет fit + transform)
- estimator (последний шаг, имеет fit + predict)

---

## 1. Transform (базовый механизм)

### Что это
Любой объект sklearn, который:
- обучается на данных (fit)
- преобразует данные (transform)

### Когда используется
Используется всегда, когда нужно:
- нормализовать данные
- заполнить пропуски
- закодировать категории
- создать новые признаки

### Примеры
- StandardScaler
- MinMaxScaler
- SimpleImputer
- OneHotEncoder
- PolynomialFeatures

---

## 2. FunctionTransformer

### Что это
Специальный инструмент, который позволяет:
- превратить обычную Python-функцию в sklearn-transformer

### Когда используется
Используется, когда:
- нужно простое кастомное преобразование
- нет смысла писать полноценный класс Transformer
- нужно быстро применить функцию к данным

### Примеры задач
- логарифмирование признаков
- ручное mapping категорий
- простые арифметические преобразования
- очистка данных

### Ограничения
- не сохраняет сложное состояние
- подходит только для простых трансформаций

---

## 3. ColumnTransformer

### Что это
Инструмент, который позволяет применять разные преобразования к разным колонкам данных.

### Когда используется
Используется, когда есть:
- числовые признаки
- категориальные признаки
- разные типы обработки для разных столбцов

### Типичный кейс
- числовые признаки → StandardScaler
- категориальные → OneHotEncoder
- специфические колонки → FunctionTransformer или кастомный transformer

### Особенности
- работает параллельно по группам колонок
- объединяет результат обратно в один набор признаков

---

## 4. Pipeline vs ColumnTransformer

### Pipeline
- последовательность шагов
- применяется ко всему набору данных

### ColumnTransformer
- параллельная обработка разных колонок
- внутри обычно используется в Pipeline как шаг

---

## 5. Custom Transformer

### Что это
Полноценный класс, который наследуется от:
- BaseEstimator
- TransformerMixin

### Когда используется
Когда:
- нужно сложное преобразование
- есть состояние (например, статистики)
- FunctionTransformer недостаточен

---

## 6. Как они комбинируются

Типичная структура sklearn-пайплайна:

1. ColumnTransformer:
   - раздельная обработка колонок
2. Pipeline:
   - объединение preprocessing + model
3. Estimator:
   - финальная модель

---

## 7. Когда что использовать

### FunctionTransformer
- простые преобразования
- быстрые эксперименты

### ColumnTransformer
- разные типы данных (числа + категории)
- реальные табличные данные

### Custom Transformer
- сложная логика
- переиспользуемые преобразования
- production-код

### Pipeline
- всегда, когда есть ML-задача

---

## Итог

- FunctionTransformer → простые функции
- ColumnTransformer → разные типы колонок
- Custom Transformer → сложная логика
- Pipeline → объединение всего процесса

Все вместе это позволяет строить чистые, безопасные и воспроизводимые ML-пайплайны в sklearn
